# sources.md Format

`sources.md` is the curated evidence trail for a topic. Topic pages should draw factual claims from here whenever possible.

## Structure

```md
# {Topic} Sources

## Knowledge

- [Resource title](https://example.com)
  What it covers. Use for: when this source should be consulted.

## Communities

- [Community name](https://example.com)
  Why it is useful. Use for: what kind of real-world feedback it provides.

## Gaps

- {Missing source, uncertainty, or evidence gap that should drive future search}
```

## Rules

- Prefer primary sources, official docs, peer-reviewed work, respected textbooks, and recognized experts.
- Annotate every entry. A bare link is not useful later.
- Surface gaps explicitly.
- Prune weak, misleading, stale, or off-mission resources.
- For health, legal, financial, and fast-changing topics, verify current claims before teaching from them.
