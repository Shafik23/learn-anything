# MISSION.md Format

`MISSION.md` lives at the root of a topic directory. It captures why the user is learning this topic. Every teaching decision should trace back to this document.

## Template

```md
# Mission: {Topic}

## Why
{1-3 sentences. The concrete real-world goal the user is chasing. What changes in their life or work when they understand this? Avoid abstract framings like "to understand X"; push for the underlying outcome.}

## Success looks like
- {A specific, observable thing the user will be able to do}
- {Another specific thing}

## Constraints
- {Time, budget, prior commitments, learning preferences, safety context, or anything else that bounds the approach}

## Out of scope
- {Adjacent topics the user explicitly does not want to chase right now}
```

## Rules

- One mission per topic directory.
- Keep the mission short enough to act as a compass.
- Revise it when the user's goal changes.
- Push back on vague missions before creating lessons.
