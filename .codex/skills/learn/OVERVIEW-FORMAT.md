# overview.md Format

`overview.md` is the direct explanation of the topic. It should answer: what is this thing, why does it matter, what are the main parts, and what is known versus unsettled?

## Template

```md
# {Topic}

{A plain-language explanation of what the topic is and why it matters.}

## What It Is

{Definition and core identity.}

## Why People Care

{Main motivations, use cases, or claims.}

## How It Works

{Mechanisms, structure, process, or mental model.}

## Important Subtopics

{The main branches the learner will study.}

## What Is Known vs Unsettled

{Separate facts, plausible claims, and open questions.}

## Key Takeaways

- {Compressed takeaway}
- {Compressed takeaway}

## Sources

- [Source](https://example.com)
```

## Rules

- `overview.md` explains the topic directly. It is not the learning plan.
- Keep the learning plan, mission, roadmap, and file index in `README.md`.
- Use links to high-trust sources for factual claims.
- Keep it readable as a standalone GitHub page.
- Update it as the learner's understanding and source base improve.
